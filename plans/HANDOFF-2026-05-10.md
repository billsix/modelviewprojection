## Quirks worth remembering

- **`docutils.utils.unescape(text, restore_backslashes=True)`** is
  required in any custom inline role that takes raw LaTeX/code text.
  Without it, `\x00` NULs slip in where backslashes were and downstream
  rendering fails opaquely (Sphinx marks the role `class="problematic"`,
  which doesn't tell you the actual cause).
- **pdflatex stale-`.aux` recovery is manual.** A failed pdflatex run
  leaves a half-written `modelviewprojection.aux` with truncated
  `\newlabel{...}{{...{...` entries. The next run reads the broken
  `.aux` and dies with "Runaway argument? File ended while scanning use
  of `\@newl@bel`" before ever reaching the actual document. `rm -rf
  book/docs/_build/latex` clears it.
- **The auto-mode classifier blocks `rm -rf` of pre-existing paths** — I
  can't wipe `_build/latex/` myself. Bill has to.
- **My local container does not have `texExpToPng`.** Build runs in the
  podman build container which installs it via meson from
  `_static/tex_exp_to_png/`. So I can write/edit the extension but I
  can't smoke-test the rendering pipeline myself; only Bill's `make
  html` exercises it. (The hash function and AST checks I CAN run.)
